Crypt Code

A simple programming language that mixes programming basics with encryption.

What is CryptCode?

CryptCode is an interpreted programming language where source files are not stored as plain readable code. You write a .ccode file, encrypt it into a .crypt file, then run the encrypted version through the CryptCode interpreter using the matching key.

Encryption methods

CryptCode currently supports three encryption methods. Each method can be assigned to a different non-empty line, and the key tells the interpreter which decryptor to use when the encrypted file runs.

  • Caesar Shifts letters using the public and private key files.
  • Reverse Flips the line content so it has to be reversed again before running.
  • AddX Applies a simple character transformation to hide the original line.

Core syntax

The language is intentionally small, so the commands are easy to read and reason about. Blocks are closed with end, and conditional branches can use else. Expressions also need spaces between values and operators, so total + i works, but total+i does not.

  • print <message or variable>
  • set <variable> to <number or expression>
  • count <var> from <start> to <end>
  • repeat <number>
  • when <var> is <number>
  • when <var> is not <number>
  • when <var> greater <number>
  • when <var> less <number>
  • when <var> mod <number> is <number>
  • else
  • end
  • fizzbuzz <start> to <end>

Supported operators include +, -, *, /, and mod.

The basic flow

A CryptCode program starts as a readable .ccode file written with simple commands like print, set, count, and when. From there, the file is encrypted into a .crypt file that can be decrypted and run by the interpreter.

  • Write Create a plain .ccode file using CryptCode commands.
  • Encrypt Convert the readable file into a .crypt file.
  • Key Add a Key: line that matches the encryption method used on each non-empty line.
  • Run Execute the encrypted file so CryptCode can decrypt the lines and interpret the program.
